How this calculator works (plain English)

What goes in: a list of items (from vision or the UI), how movers get in and out at pickup and dropoff (ground, stairs, or elevator, plus floor count), and optional travel minutes between locations.

What comes out: how long the move should take, how many movers we suggest, and a price (number of movers × time × hourly rate, plus tax).

Where the numbers live: most per-item minutes come from the catalog or database. Rules for stairs, elevators, crew size, and tax are fixed values in code (see the constants table at the bottom).

Steps in order

1) Time per item (minutes)
   load_minutes = catalog base time (+ extra if customer asked to disassemble)
   unload_minutes = load_minutes × unload factor (unload is often quicker)
   item_total_minutes = load_minutes + unload_minutes

2) Time for the whole job (before travel)
   Add up all items (boxes grouped together may get a discount).
   If you send more movers, the job finishes sooner—but extra movers do not cut time in half.
   Pickup/dropoff stairs: multiply job time by (1 + stair penalty for pickup + stair penalty for dropoff).
   Elevator: add extra minutes for elevator trips (separate from the multiply above).

3) How many movers?
   Try 2, 3, … up to how many seats fit in the truck.
   For each option, compute job time and cost; pick the option that balances speed vs price.
   Small jobs favor cheaper crew; bigger jobs still care about cost but a bit more about time.

4) Price
   total_minutes = job time + travel
   price = total_minutes × number of movers × dollars per minute × (1 + tax rate)

Stairs vs elevator: stairs make the whole move take longer (a percent-style bump per floor). Elevator adds extra minutes for riding and waiting (per trip), not as a percent of item time.

compute_base_item_time (active only)

  1. base = baseTime[size] SQL
  2. If disassemble flag → base += disassemblyAdder[size] SQL + USER
  3. load_time = base
  4. unload_time = base × unload_ratio HARDCODED ratios; hints from SQL stackable / disassemblyAdder
  5. total = max(load + unload, 2.0)

Task fields (after build)

id, name, p, load_time, unload_time, size, stackable, stackableSavings, weight, volume — batched tasks add count, is_batch.

Stackable batching

Group by (name, size) when stackable and stackableSavings > 0. Combined times × (1 − savings%/100). SQL

Weight/volume: from catalog if missing — vehicle selection only, not time adders.

Stairs (multiplicative)

  • Per leg: delta = 0.06 + 0.035 × floors (floors capped at 50)
  • adjusted_time = base_time × (1 + pickup_delta + dropoff_delta)

Elevator (additive wall-clock)

  • trip_time = 1.5 + 0.08 × floors
  • trips = ceil(num_tasks / 2.0) — num_tasks = len(tasks) after batching
  • Both legs summed; not proportional to item complexity
  • If elevator on either leg: effective_teams = min(teams, 1.5)
access.type, access.floors USER · STAIR_*, ELEVATOR_*, AVG_ITEMS_PER_ELEVATOR_LOAD HARDCODED (not catalog elevatorFit)

6. Load / unload reporting & pricing

The UI shows separate loading and unloading minutes by splitting total job time in proportion.

After chosen crew and travel:

  • labor_minutes = job time for recommended crew
  • base_load / base_unload = sum of task load_time / unload_time
  • load_ratio = base_load / (base_load + base_unload) (fallback ~0.606)
  • loadingTime = labor_minutes × load_ratio
  • unloadingTime = labor_minutes × (1 − load_ratio)

Stairs friction and elevator apply at job level; UI split is proportional to pre-friction task totals.

Pricing: base_price = total_minutes × recommended_m × WAGE_RATE; GST 5%; min/max hours from Vision_Agent_Hour_Range_Mapping.csv.

8. Catalog fields not used by V9 Python

Do not port to C# unless product spec changes

  • heavyAdder — removed from Python; pipeline always sets weight from catalog
  • bulkyAdder — never referenced in calculator
  • elevatorFit — never referenced; elevator trips use AVG_ITEMS_PER_ELEVATOR_LOAD
  • twoPersonRequired — never referenced; no per-item q on tasks; crew is job-level only

JSON/DB may still store these for other systems.
