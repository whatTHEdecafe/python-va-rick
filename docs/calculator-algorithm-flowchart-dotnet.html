MoveasyBackend · BookingService · SQL VisionItems

Primary crew API: IdentifyLabourCount (~L1474) · Separate booking-time path: CalculateTimeForSize / FetchVisionBookingDetails

SQL VisionItems / Companies USER API request CONFIG appsettings MovingCalculator HARDCODED Code constants / fleet

Two algorithms in one service

A — IdentifyLabourCount (Vision API SuggestedLabour): task list + parallelism + flat % friction + crew scoring. Partially V9-aligned via config.

B — FetchVisionBookingDetails → CalculateTimeForSize: per-item time summed for booking totals; stairs/elevator = floors × DB adder per flight/ride; stackable from VisionItems.StackableSavingsPct. Does not pick crew size.

Per-item time (Path A)

  1. baseMin from VisionItems.BaseTime{S,M,L}Min SQL
  2. +5 if IsDisassembleRequired HARDCODED (not DB disassembly columns)
  3. +5 if size large HARDCODED bulky proxy
  4. × LoadUnloadMultiplier (default 1.2) CONFIG — single multiplier, not separate load/unload
  5. max(itemTime, 2)

Stackable (Path A)

IsStackable = name contains box/bag OR size small HARDCODED. Grouped items: sum time × 0.70 (30% savings flat). Does not use StackableSavingsPct from SQL.

MoveTask fields

Name, BaseTimeMinutes, CrewRequired, IsStackable — CrewRequired is set (large or time>15 → 2) but not used in CalculateJobTime.

Defaults (appsettings MovingCalculator)

  • Stairs: +0.20 once if either leg mentions stairs
  • Elevator: +0.10 once if either leg mentions elevator
  • Long carry: +0.10 if string contains long

Not the same as Python V9

  • Python: stairs = per-floor delta (0.06 + 0.035×floors) summed into multiplier
  • Python: elevator = additive trip minutes after multiply; cap teams at 1.5
  • .NET Path A: elevator is only a +10% multiplier; no ELEVATOR_PARALLELISM_CAP
  • .NET Path A: CalculateMoverRequest has no floor fields

Travel & cost

  • travelTime = TotalDistanceKm × 1.5 + 30 HARDCODED
  • Scoring uses JobTimeMinutes only (labor) — aligned with V9 intent
  • LaborCost = TotalTimeMinutes × m × wagePerMin (includes travel in cost line)
  • EstimatedCostIncludingGST = LaborCost × (1 + GstRate)

No forced_movers on CalculateMoverRequest (Python has optional override).

Path B — Vision booking totals (per item)

FetchVisionBookingDetails → GetItemsDetails → CalculateTimeForSize

flowchart TD
    Items["VisionAnalyzeItem list"] --> Vi["VisionItems from SQL"]
    Vi --> Calc["CalculateTimeForSize per item"]
    Calc --> Base["baseTime + disassembly from DB"]
    Base --> Pick["pickup: floors x Stairs/Elevator adder"]
    Pick --> Drop["dropoff: floors x adder"]
    Drop --> Stack{"StackableFlag YES?"}
    Stack -->|yes| Save["qty x perItem x 1 - savings%"]
    Stack -->|no| Lin["qty x perItem"]
    Save --> Sum["totalTime += item"]
    Lin --> Sum

Per-item formula (Path B)

perItem = (BaseTime + DisassemblyAdder) × Company.LoadUnloadmultiplier + pickupFloors×adder + dropoffFloors×adder

Stairs/elevator adders: StairsAdderPerFlight{S,M,L}Min, ElevatorAdderPerRide{S,M,L}Min SQL. Mediums: MainFloor, Stairs, Elevator.

Stackable: uses StackableFlag + StackableSavingsPct from DB SQL.
